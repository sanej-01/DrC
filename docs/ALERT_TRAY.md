# Alert Tray — Phase 6.4

Manager notifications for score drops and other events.

---

## Overview

**Purpose:** Alerts managers to significant events (score drops, new feedback, disputes) requiring attention.

**Access:** Managers+ only

**Visibility:** Workspace-scoped; managers only

---

## Data Model

### alerts Table

```sql
CREATE TABLE alerts (
  id UUID PRIMARY KEY,
  workspace_id UUID NOT NULL (FK),
  manager_id UUID NOT NULL (FK),
  developer_id UUID NOT NULL (FK),
  alert_type alert_type ('score_drop' | 'new_feedback' | 'dispute_filed'),
  title TEXT,
  description TEXT,
  metric_name TEXT,
  metric_old_value DECIMAL(5,2),
  metric_new_value DECIMAL(5,2),
  threshold_delta DECIMAL(5,2),
  pr_id UUID (FK),
  coaching_card_id UUID (FK),
  dispute_id UUID (FK),
  status alert_status ('active' | 'snoozed' | 'dismissed'),
  snoozed_until TIMESTAMP,
  snoozed_by UUID (FK),
  dismissed_at TIMESTAMP,
  dismissed_by UUID (FK),
  dismissal_reason TEXT,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

### Alert Types

| Type | Trigger | Example |
|------|---------|---------|
| `score_drop` | Developer score decreases >5 points | "Score drop: code_quality from 85 to 78" |
| `new_feedback` | New coaching card created | "New coaching: Alice received IMPROVE feedback" |
| `dispute_filed` | Developer disputes score | "Alice disputed PR #42 score" |

### Alert States

| Status | Meaning |
|--------|---------|
| `active` | Unread, requires attention |
| `snoozed` | Hidden until `snoozed_until` time |
| `dismissed` | Manually hidden, won't reappear |

## API Endpoints

### GET /api/manager/alerts

Fetch active and snoozed alerts.

**Query params:**
- `workspace_id` (required)
- `limit` (default 10)

**Response:**
```json
{
  "alerts": [
    {
      "id": "uuid",
      "alert_type": "score_drop",
      "title": "Score drop detected: code_quality",
      "description": "code_quality decreased from 88 to 78 (-10.0)",
      "metric_name": "code_quality",
      "metric_old_value": 88,
      "metric_new_value": 78,
      "threshold_delta": -10,
      "status": "active",
      "created_at": "2026-07-09T14:30:00Z",
      "developer": {
        "display_name": "Alice",
        "github_handle": "alice-dev"
      }
    }
  ]
}
```

**Behavior:**
- Expired snooze alerts (snoozed_until < now) returned as "active"
- Ordered by created_at DESC (newest first)

### POST /api/manager/alerts

Snooze or dismiss an alert.

**Body:**
```json
{
  "action": "snooze" | "dismiss",
  "alert_id": "uuid",
  "reason": "optional reason",
  "minutes": 1440
}
```

**Actions:**

**snooze:**
- Sets status = "snoozed"
- Sets snoozed_until = now + minutes
- Default 1440 minutes (24 hours)
- Alert reappears as "active" when snooze expires

**dismiss:**
- Sets status = "dismissed"
- Alert never reappears
- Dismissal reason logged for audit

**Response:** Updated alert object

## Score Drop Detection

Score drops ≥5 points trigger alerts:

```
old_score = 85 (code_quality)
new_score = 78
delta = 78 - 85 = -7

if delta <= -5:
  create alert(
    type: 'score_drop',
    metric_name: 'code_quality',
    metric_old_value: 85,
    metric_new_value: 78,
    threshold_delta: -7
  )
```

**Triggered by:**
- New PR scoring (if score ≥5 lower than previous)
- Aggregate recomputation (if 30d average drops)

**One per developer per metric:** Prevents duplicate alerts for same issue.

## Snooze & Dismiss

### Snooze Behavior

- Alert hidden from tray for duration (default 24 hours)
- Timer stored in `snoozed_until`
- When timer expires, alert auto-reappears as "active"
- Manager can snooze multiple times

### Dismiss Behavior

- Alert permanently hidden
- Status = "dismissed", never changes
- Optional reason logged ("Already handled", "Not relevant", etc.)
- Appears in audit log for compliance

---

## Alert Lifecycle

```
1. CREATE: Alert generated by score drop or event
   status = 'active'
   created_at = now
   
2. ACTIVE: Manager sees in tray
   can snooze (→ snoozed) or dismiss (→ dismissed)
   
3a. SNOOZED: Hidden for duration
   snoozed_until = now + X minutes
   snoozed_by = manager_id
   (Automatically returns to active when snoozed_until < now)
   
3b. DISMISSED: Permanently hidden
   dismissed_at = now
   dismissed_by = manager_id
   dismissal_reason = optional text
```

## Performance

**Queries per load:**
- 1 query to fetch alerts (limited to 10 by default)
- Indexes on (workspace_id, status), snoozed_until

**Polling:** Every 30 seconds (configurable)

**Caching:** None (real-time alerts important)

## RLS & Security

**Manager view policy:** Can see alerts for their workspace
```sql
workspace_id IN (
  SELECT workspace_id FROM workspace_members
  WHERE user_id = auth.uid() AND role IN ('manager', 'admin')
)
```

**Developer deny policy:** `FOR ALL USING (false)` — developers never see alerts

**Cross-workspace isolation:** Workspace_id filter enforced

## Audit Trail

### alert_actions_log Table

Tracks all alert actions:

```sql
CREATE TABLE alert_actions_log (
  id UUID PRIMARY KEY,
  alert_id UUID (FK),
  action TEXT ('created' | 'snoozed' | 'dismissed' | 'expired'),
  performed_by UUID (FK to users),
  reason TEXT,
  created_at TIMESTAMP
);
```

Used for:
- Compliance audits ("who dismissed alerts?")
- Analytics ("most common dismiss reasons?")
- Debugging alert behavior
